# 05 大模型驱动智能操作系统原型

本项目是对《操作系统》课程设计基础实验的扩展，使用 `Python + DeepSeek API + 本地前端` 构建一个“大模型驱动的智能操作系统实验原型”。

## 功能概述

系统支持以下演示流程：

1. 输入一句自然语言实验请求
2. 由大模型先进行边界判断
3. 如果请求符合课程实验范围，则解析为结构化任务
4. 自动生成对应基础模块的输入参数文件
5. 自动调用基础模块程序执行
6. 展示原始执行结果
7. 再由大模型输出实验分析结论

如果输入内容不属于课程实验框架，系统会提示：

- 不符合当前实验原型支持范围
- 需要重新输入

## 已接入基础模块

- `01_处理机调度`
- `02_内存管理`
- `03_进程同步与并发控制`
- `04_文件系统`

## 目录结构

- `app.py`：后端入口
- `core/`：边界代理、DeepSeek 客户端、任务解析、执行器
- `web/`：前端页面
- `runtime/`：运行时动态生成的输入文件和缓存结果
- `start.bat`：Windows 一键启动脚本
- `config.example.json`：配置模板

## 前置要求

1. 已安装 Python 3.10+
2. 已安装并可进入 WSL Ubuntu
3. 4 个基础模块已完成编译，或者能在 WSL 中通过 `make` 自动编译
4. 你有可用的 `DeepSeek API Key`

## 配置方式

复制一份配置文件：

```text
config.example.json -> config.json
```

然后填写：

- `deepseek_api_key`
- `wsl_distro`

其中 `wsl_distro` 建议填写你的 Ubuntu 发行版名称，例如：

```json
"wsl_distro": "Ubuntu"
```

## 启动方式

直接双击：

```text
start.bat
```

或在 PowerShell 中执行：

```powershell
.\start.bat
```

启动后浏览器访问：

```text
http://127.0.0.1:8899
```

## 支持的自然语言方向

### 处理机调度

- 创建若干进程并指定到达时间、服务时间、优先级
- 指定算法：`FCFS`、`SJF`、`RR`、`Priority`
- 指定时间片
- 要求算法对比

### 内存管理

- 页面置换：`FIFO`、`LRU`
- 动态分区：`FF`、`BF`
- 指定页框数、访问序列、总内存、分配回收操作

### 进程同步与并发控制

- 生产者消费者
- 读者写者
- 哲学家进餐

### 文件系统

- 目录创建、切换
- 文件创建、写入、读取、删除
- 状态查看

## 当前边界

本原型只支持“操作系统课程实验相关请求”，例如：

- 调度算法模拟
- 内存管理实验
- 并发同步实验
- 简易文件系统实验
- 对实验结果做解释分析

不支持：

- 闲聊
- 通用百科问答
- 与本课程设计无关的编程任务
- 要求系统执行真实危险操作

## 说明

本项目强调的是：

- 大模型驱动交互
- 自然语言到实验参数的自动转换
- 课程实验结果的自动解释

它不是一个真正替代操作系统内核的系统，而是一个“智能实验原型”。
